# Survey Question Designer

Write surveys that produce clear, usable signal instead of vague feedback. Reduce bias and improve question flow.

## Quick Reference

**Key Insight:** Every question should inform a decision. Avoid questions that only satisfy curiosity. Use the right question type for the data you need.

## When to Activate

Activate when the user asks to:
- create surveys or feedback forms
- design marketing or customer research
- improve question flow or reduce bias

## Workflow

### Step 1: Define Goal

Identify:
- primary decision the survey informs
- audience and channel
- required data types

### Step 2: Question Types

Reference [references/question-types.md](./references/question-types.md):

**Screener** – Filter respondents into the right audience

**Multiple Choice** – Clean comparison, segmentation-friendly answers

**Rating** – Satisfaction, importance, confidence; avoid stacking too many

**Open Response** – When language matters and you want customer voice

**Follow-Up Probe** – Only when prior answer creates meaningful branch

### Step 3: Question Flow

Order questions to:
- screen first
- start easy, build to harder
- avoid leading or loaded language
- limit length (drop questions that don't inform the goal)

### Step 4: Bias Reduction

- Avoid double-barreled questions
- Use neutral wording
- Provide balanced options
- Avoid leading questions

### Step 5: Output Structure

Produce:
- survey goal and decision to inform
- question list with type and rationale
- flow and branching logic
- bias-reduction notes

## Quality Gates

Before finalizing, verify:
- [ ] Every question informs the goal
- [ ] Question types match data needs
- [ ] Flow is logical
- [ ] Bias is minimized
