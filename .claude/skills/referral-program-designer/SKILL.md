# Referral Program Designer

Build referral systems people understand and want to use. Keep incentives clear and friction low.

## Quick Reference

**Key Insight:** Referral programs work when both parties get value and the ask is easy. Over-complicated rules or delayed rewards kill participation.

## When to Activate

Activate when the user asks to:
- launch or improve a referral program
- design referral incentives
- plan referral triggers and copy
- set operational rules for referrals

## Workflow

### Step 1: Define Goal

Identify:
- primary goal (acquisition, retention, both)
- target referrer (customer, creator, partner)
- budget or reward constraint

### Step 2: Reward Model

Reference [references/reward-models.md](./references/reward-models.md):

**Give-Get** – Both parties receive value. Works when sharing is low-friction.

**One-Sided Reward** – Referrer does most of the work; buyer incentive less relevant.

**Tiered Reward** – Increasing payout over time to motivate repeat referrals.

**Milestone Reward** – One-time reward for high-effort referrers.

### Step 3: Triggers and Copy

Design:
- when to ask (post-success, post-purchase, milestone)
- copy hooks (clear benefit, social proof, urgency)
- sharing mechanics (link, code, invite flow)

### Step 4: Operational Rules

Define:
- eligibility (who can refer)
- reward timing (immediate, on conversion, on payout)
- fraud prevention
- caps or limits

### Step 5: Output Structure

Produce:
- reward model and structure
- trigger points and copy
- operational rules
- success metrics

## Quality Gates

Before finalizing, verify:
- [ ] Incentive is clear to both parties
- [ ] Friction is minimized
- [ ] Operational rules are enforceable
- [ ] Fraud prevention is considered
